# Module 5: Putting It All Together

**Duration**: ~35 min total · 3 lessons

You have all the pieces. Module 5 is where they become a repeatable workflow — something you can run on any project, not just this workshop exercise.

## Lessons

| # | Title | Focus | Time |
|---|-------|-------|------|
| L5.1 | [The Full Spec-Driven Simulation](/module5/L5-1-full-workflow) | End-to-end run: spec → plan → tasks → code → review | ~15 min |
| L5.2 | [Real-World Use Cases](/module5/L5-2-use-cases) | Three project templates + kickoff checklist | ~10 min |
| L5.3 | [Progressive Automation](/module5/L5-3-automation) | Level 1 mandatory automation, Level 2 optional loop | ~10 min |

## What You'll Build

By the end of Module 5, you will have:
- Completed one **full spec-driven cycle** on a fresh feature, without assistance.
- A **kickoff checklist** that you can use on any new project.
- A starting **Level 1 automation** — an auto Code Reviewer that runs on every generated implementation.
- A personal answer to: "When would I skip this? When would I not?"

## Module Outcome

You leave with a methodology, not a trick. The spec-driven workflow is a practice that scales from a workshop feature to a team-managed product. Module 5 gives you the map for how to use it beyond today.
